Task: Categorize Components
Categorize all components into work pipelines for further evaluation, selection, and prioritization.
Disciplines: Product Portfolio Management
Extends: Categorize Components
Purpose
To ensure that components are processed in a consistent manner, using processes and techniques adapted to a particular work pipeline, so that portfolio-level balancing metrics and targets can be applied.

      Main Description

      This is the task where all qualified components are assigned to a specific work pipeline to be evaluated, selected, and prioritized. These can be among the work pipelines:

      • Production support
      • Application maintenance
      • Strategic projects

      Each pipeline represents work of a different type and contains components that share common characteristics. Moreover, each pipeline is aligned with the organization's strategic planning process and is regulated by a predefined set of criteria and metrics.

      Work of different types warrant different levels of process and controls. For example, the data definition requirement for a support request differs from that of a strategic project.

      Re-categorize components as needed during the process, as more knowledge is built around each component.

      Steps
      Assess incoming components
      Revisit and reassess components based on the defined criteria and filters for the work pipeline.
      Assign incoming components
      Assign components to a specific pipeline, such as a product line. That assignment entails the component to be treated in a similar way as the other components in that pipeline.
