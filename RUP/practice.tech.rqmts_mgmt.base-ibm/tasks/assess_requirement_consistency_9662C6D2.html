Task: Assess Requirements Consistency
Perform a periodic assessment of consistency across requirements
Disciplines: Requirements
Purpose
The purpose of this task is to assess the requirements consistency through periodic requirements traceability.
Relationships
RolesPrimary Performer:
  • Analyst
Additional Performers:
InputsMandatory:
  • Requirements Attributes
  • Requirements Traceability
    		Optional:
    • None
    Outputs
    • Requirements Attributes
    • Requirements Traceability
      Steps
      Assess requirements traceability

      Perform requirements assessment by retracing requirements with project work products.

      • If there is any issue that is immediately solvable, fix the issue, otherwise record the defect in accordance with the defect logging practice.
      • If a change is required, propose the change request through the change management practice.
      Assess requirement attributes
      As the project evolves, requirement attributes tend to change. Check whether the attributes reflect the current project information.
      Key Considerations
      The requirements tend to degrade over time as they evolve. Periodic assessment can be performed once per iteration or milestone.
