Main Description

Methods of requirements packaging structures include:

  • One package per requirement type
  • One package for each set of related functional requirements, plus an additional package for system-wide requirements

More granular packages can be created within these packages as needed. For small projects, it might be sufficient to create project folders and subfolders for the packages on a secure shared drive. For larger and more complex projects, it is beneficial to define the packages within a requirements management tool.

See the Tool Mentor: Packaging Requirements in RequisitePro for information on how to use the tool to package requirements.
