Tool Mentor: How to establish and verify traceability with RequisitePro
Use IBM Rational RequisitePro to establish and verify traceability.
Tool: Rational RequisitePro

Related Elements
  • Manage Changing Requirements
Main Description

RequisitePro facilitates tracing between items stored in the requirements repository to ensure consistency between requirements, use cases, and test cases.

The three main steps in establishing and verifying traceability are assigning the traceability, creating the traceability matrix, and verifying the traceability.

Assigning the traceability between the requirements and use cases or test cases:

  1. From the navigation tree, right click the requirement on the navigation tree that you want to trace
  2. Select Properties
  3. The Requirement Properties box is displayed
  4. Select the Traceability Tab
  5. On the right side of the Form box, click Add to add the other project item(s) that this requirement is traced from
  6. Select the project item(s) that this requirement is traced from
  7. Click OK
  8. On the right side of the To box, click Add to add the other project item(s) that this requirement traces to
  9. Select the project item(s) that this requirement traces to
  10. Click OK
  11. Repeat Step 1 for each requirement until completed

See also Rational RequisitePro online Help > Contents and Index > Creating and deleting requirement traceability information>

Creating a traceability matrix:

  1. From the navigation tree, right click the project name or package name
  2. Select New > View
  3. Enter the traceability Name
  4. Enter the traceability Description
  5. Change View type to Traceability Matrix
  6. In the Row Requirement Type drop-down list, select the item that you want to trace, such as FEAT: Feature
  7. In Column Requirement Type, select the item that you want to trace with, such as TC: Test Case
  8. Click OK
  9. The matrix will be shown on the working panel.
  10. Select File > Save View to save this traceability matrix
  11. Repeat Step 1 for other desired traceability

See also the Rational RequisitePro online Help > Contents and Index > Creating traceability relationships in a view>

Verifying traceability:

  1. From the navigation tree, double-click the traceability matrix that you have created
  2. The traceability matrix will be shown on the working panel
  3. The arrow in a cell shows the traceability relationship between the project item in that row and another project item in that column
  4. The direction of the arrow shows the direction of the traceability: the head of the arrow shows the project that item is traced to, and the end of the arrow shows the project that item is traced from.
  5. From the display of the matrix, a row or a column that has no arrow means that it has no traceability, so the analyst needs to explore and investigate this item.
