Tool Mentor: How to assign attributes with RequisitePro
Use IBM Rational RequisitePro to assign attributes of the requirements.
Tool: Rational RequisitePro
Relationships
Related Elements
  • Manage Changing Requirements
Main Description

IBM® Rational® RequisitePro® helps you to store the project's requirements. To initialize your requirement repository, RequisitePro provides you with various project templates, such as RUP template, Use Case Template, Traditional Template.

To create a new requirement and assign attributes to a requirement:

  1. Select File > New > Requirement
  2. The New Requirement Properties box is displayed
  3. Fill in the following information in the General tab
    • Type: Select the proper type from the Requirement Type drop-down list
    • Name: Type the short name of the requirement
    • Description: Type the description of the requirement
    • Package: Select the proper package by clicking the Browse button
  4. Select the Attributes tab to fill in the following information
    • Priority: Select the proper priority from the drop-down list, for which the possible values are Low, Medium, or High
    • Status: Select the proper status (Proposed, Approved, Incorporated, or Validation)
    • Difficulty: Select the proper difficulty level (Low, Medium, or High)
    • Stability: Select the proper stability value (Low, Medium, or High)
    • Assigned To: Type the name of the person who is responsible for this requirement
    • Origin: Select the proper origin value (Customers, Development, or Executive Committee)
    • Obsolete: Select the proper status (Yes or No)

See also Rational RequisitePro online Help > Contents and Index >Assigning values to requirement attributes

Note: If the attribute type or requirement type is not available in your project template, you can add them by doing the following:

  1. Select File > Project Administration > Properties
  2. The New Requirement Properties box is displayed
  3. Do the following information in the Requirement Type or Attributes tab:
  4. Click Add to add a new type
  5. Fill in all necessary information and click OK

With the initial version of requirement, the version number in the Revision tab will automatically start at 1.0001
