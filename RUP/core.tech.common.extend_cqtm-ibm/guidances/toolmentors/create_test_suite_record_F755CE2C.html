Creating a test suite record

A test suite record contains a ordered list of configured test case records. Before creating a test suite record, determine which iteration and configuration records will apply to the test suite record. Before you can create a test suite record, you must have already created an asset registry record.
Note: You must ensure that the test suite record specifies the same platform as that of the Rational ClearQuest Client or Rational ClearQuest Client for Eclipse on which the test suite record will be executed.
To create a test suite record:
  1. Log in to a database that supports Rational ClearQuest Test Manager.
  2. Using a supported Rational ClearQuest client:
    1. From a Rational ClearQuest for Windows client, click Actions > New and from the Choose Record Type window, click TMTestSuite and then click OK.
    2. From a Rational ClearQuest Web client, click New and then select TMTestSuite and click Go.
  3. Click the Main page. The Main page displays general information.
    1. Type a headline for the test suite in the Headline field.
    2. Click to select an Owner for the test suite. Note, in Rational ClearQuest Test Manager the Owner has no additional privileges.
    3. Type an optional description in the Description field.
    4. Click to select an Asset Registry.
    5. Click Add and use the Browse Record Type window to select an associated iteration record.
  4. Click the Execution page. The Execution page displays the associated configuration, configured test case, and suite log records.
    1. Click Select and use the Browse Record Type window to select an associated configuration record.
    2. When a test suite record is created, the lists for Configured Test Cases, Execution Order of Configured Test Cases, and Test Suite Logs are empty. You must use a Rational ClearQuest Client or Rational ClearQuest Client for Eclipse to add an ordered list of associations for configured test case records.
    3. After a test suite record is executed, the associated suite log record is displayed in the Test Suite Logs list.
  5. Click the History page. The History page displays an audit trail for changes to the record.
  6. Click the Notes page. The Notes page displays annotations to the record.
  7. In the Rational ClearQuest for Windows client, click OK. In the Rational ClearQuest Web client, click Save.
Use the queries supplied by Rational ClearQuest Test Manager to locate a specific test suite record for modification or state changes.
