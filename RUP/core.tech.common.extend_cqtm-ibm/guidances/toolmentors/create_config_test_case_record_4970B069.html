Main Description

Creating a configured test case record

Before creating a configured test case record, you must create the iteration and configuration records associated with the configured test case. A specific configured test case record can be related to only one test case record and one configuration record. Before you can create a configured test case record, you must have already created a parent test case record.
Note: The configuration record must specify the same platform as that of the Rational ClearQuest Client or Rational ClearQuest Client for Eclipse where the configured test case record will be executed.
To create a configured test case record:
  1. Log in to database that supports Rational ClearQuest Test Manager.
  2. Using a supported Rational ClearQuest client:
    1. From a Rational ClearQuest for Windows client, click Actions > New and from the Choose Record Type window, click TMConfiguedTestCase and then click OK.
    2. From a Rational ClearQuest Web client, click New and then select TMConfiguedTestCase and click Go.
  3. From the configured test case, click the Main page. The Main page displays general information.
    1. Type a headline in the Headline field for the configured test case.
    2. Click to select a Priority.
    3. Click to select an Owner for the configured test case. Note, in Rational ClearQuest Test Manager, the record Owner has no additional privileges.
    4. Click Select and use the Browse Record Type window to select an associated configuration record.
    5. Click Select and use the Browse Record Type window to select an associated parent test case record.
    6. Click Select and use the Browse Record Type window to select an associated iteration record.
  4. Click the Execution page. The Execution page displays the associated test script and test log records.
    1. If the parent test case contains the file location reference for the test script, click Copy from Test Case. If you want to create a new association or associate a different test script with the configured test case record, you must use a Rational ClearQuest Client or aRational ClearQuest Client for Eclipse.
    2. Initially, there are no references to test log records. Test log references appear only after the configured test case record is executed and the test results are promoted.
  5. Click the History page. The History page displays an audit trail for changes to the record.
  6. Click the Notes page. The Notes page displays your annotations.
  7. Click OK to save the record.
Use the queries supplied by Rational ClearQuest Test Manager to locate a specific configured test case record for modifications or state changes.
