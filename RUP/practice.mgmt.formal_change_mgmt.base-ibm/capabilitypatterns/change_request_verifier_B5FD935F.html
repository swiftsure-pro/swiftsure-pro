Main Description

This role represents the individual on the development team who is responsible for verifying that a resolved change request does in fact satisfy the intent of the request. This is not a new role, as anyone playing this role will also be playing another role while performing this task. For example:

  • For a change request that implements a new feature or scenario, the Request Verifier would also be a Tester that tests the system to verify the implementation.
  • For a change request that adds a new requirement, the Request Verifier would also be an Analyst that performs a peer review.

Because the skills required for this role depend on which artifacts are affected, it is convenient to capture this responsibility in a generic role.

What are the responsibilities of the Change Request Verifier?

1. Develop test procedures.
2. Schedule and run tests.
3. Document the test results.
4. Use the test results as part of the change documentation.
5. Notify all stakeholders who are impacted by the change.
