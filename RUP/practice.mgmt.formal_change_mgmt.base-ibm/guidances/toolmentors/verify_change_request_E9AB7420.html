Tool Mentor: Verifying a Change Request Using Rational ClearQuest
This tool mentor describes how to verify a change request using Rational ClearQuest.
Tool: Rational ClearQuest
Relationships
Related Elements
  • Verify Change Request
Main Description

Tool Steps

  1. Run a query to identify change requests that need to be Verified (for example: "Resolved Defects").
  2. Select the change request by marking its line on the result set grid. Examine the change.
  3. Perform the required tests to evaluate the resolution. This operation can be done manually or by automation, and can be performed on a single defect, several defects, or using a full regression test.
  4. Depending on the test result, select the action Verify_pass or Verify_fail.
  5. Enter information in the mandatory fields. Optionally, enter information in additional optional fields as required (for example: fail reason).
  6. If required, add attachment(s) in the Attachment tab.
  7. Click OK.

Hint: If you have several defects to verify, you can mark all of those that passed the test and perform a single Verify_pass operation.
