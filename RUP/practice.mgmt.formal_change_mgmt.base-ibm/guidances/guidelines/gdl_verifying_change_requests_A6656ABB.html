Guideline: Verifying a Change Request
This guideline provides details on how to verify a change request.
Relationships
Related Elements
  • Verify Change Request
Main Description

When you verify a change request, you ensure that the development activities associated with the change request has been completed, and that the resulting change set satisfies the intent of the change request.

Depending on the type of items that were changed, this verification may take the form of a formal review of content, functional testing against original requirements, and so on. The essence of this task is to review all of the associated objective evidence (for example, test logs, formal review records, inspection reports, and so on) to validate that the work has been completed, and that the change request can be dispositioned to the next state.
