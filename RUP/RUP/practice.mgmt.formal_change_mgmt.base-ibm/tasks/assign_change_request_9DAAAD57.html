Task: Assign Change Request
This task assigns a reviewed change request to an individual for implementation in a specified iteration.
Disciplines: Configuration and Change Management
Purpose

The purpose of this task is to assign a change request to an individual for implementation in a specified iteration. This task is performed as part of the iteration planning.

      Main Description

      This task assigns a change request to a resolver for implementation. Work is assigned, estimated, and tracked. When the task is complete, it is set to Resolved, as with any other work item.

      For the purposes of this task, resources refer to the people who might carry out the task, and not to any equipment that might be required for the task.

      Key Considerations

      Determine if there other similar or related tasks that need to be considered. Consider what resources you have available.

      • Are there are other similar or related tasks that need to be considered for planning purposes for the target iteration (for example, you can't implement one without the other)?
      • What resources do you have available, and who is the best resource for the task?
      • Is the level of effort understood and agreed to?
      • What iteration does the change request need to be associated with?
      • Who should be working on which tasks and when?
      • Do you have the correct number of resources with the correct skill set to accomplish the scope of work that your project requires?
      • Are you expecting a resource to work on a task at a time when that resource will not be available to work (for example, when the resource will be on vacation)?
      • Have you assigned a resource to so many tasks that you have exceeded the capacity of the resource to work? Is the resource over-allocated?
