Tool Mentor: Import Test Documents Using IBM Rational Manual Tester
This tool mentor describes how to use IBM Rational Manual Tester to import test documents.
Tool: IBM Rational Manual Tester

Main Description

Overview

In the process of implementing tests, you will most likely develop test scripts from within IBM® Rational® Manual Tester's script editor. You may also reuse parts of scripts from the Reuse view (see the Tool Mentor: Reuse Content from a Test Script Using IBM Rational Manual Tester). Another convenient way to create scripts is to import test documents directly into Rational Manual Tester, and then having Rational Manual Tester create a test script out of the imported document. In this way, you can avoid having to recreate tests written in Microsoft® Word or Excel.

Tool Steps

To import a test document using Rational Manual Tester, perform the following steps:

  1. Select File > Import
  2. In the Rational Manual Tester Import wizard, select the file type you want to import
  3. Select the path where you want the processed file to be output
  4. Add the file you want to import and click Finish

Rational Manual Tester handles the elements of test documents differently, depending on which type of document format you are importing.

For the name of the manual test script created out of this import process, Rational Manual Tester uses the filename of the document you import and appends a .rmt extension. For example, if you import an Excel worksheet named TestingLoginApp.xls, Manual Tester names the manual test script TestingLoginApp.rmt.

For more information on how to work with file importing, see the topic Importing Test Documents in the Rational Manual Tester online Help.
