Role: Change Request Implementer
The Change Request Implementer is the person responsible for implementing the change.
Role Sets: Configuration and Change Management
Relationships
Primary Performs
  • Resolve Change Request
Modifies
  • Change Request
    Main Description

    This role represents the individual on the development team who is responsible for implementing a change request. This is not really a new role, as anyone playing this role will also be playing another role on the development team. Because the skills required will depend upon which particular artifacts are affected, it is convenient to capture this responsibility in a generic role.

    What are the responsibilities of the Change Control Implementer?

    The change control implementer is responsible for the following:

    • Obtain approvals from the project sponsor before the change request is scheduled.
    • Review the testing results from the change development.
    • Develop and prepare the change for production.
    Staffing
    SkillsThe particular skills required to implement a change request will depend upon the artifact affected.
    Assignment ApproachesAssign someone with the appropriate skills, as defined on the other process roles.
    Key Considerations
    On large projects, this role is typically assigned to those with a business analyst or a developer skill set.
