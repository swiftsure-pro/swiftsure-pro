Capability Pattern: Configuration & Change Management
This capability pattern covers the activities and workflow for the Configuration and Change Management discipline.
DescriptionWork Breakdown StructureTeam AllocationWork Product Usage
Purpose

To explain the work involved in the Configuration and Change Management discipline, the activities, tasks and work products are organized into a capability pattern for the discipline.

Description

Each activity in this capability pattern represents a high-level goal that needs to be achieved to perform effective configuration and change management. Activities such as planning the project configuration & change control and creating project configuration management environments are performed at the start of a project. The rest are performed on an on-going basis throughout the project life-cycle.

Usage Notes

Decide How to Perform the Workflow

The following decisions should be made regarding the Configuration & Change Management discipline's workflow:

  • Decide how to perform the workflow by looking at the activities in this workflow. Study the diagram, with its guard conditions, and the guidelines below. Decide which activities to perform and in which order.
  • Decide what parts of the Configuration & Change Management activities to perform. For example, the Activity: Manage Change Requests is performed only if you are going to manage the change requests in a systematic way.
  • Decide when, during the project lifecycle, to introduce each part of the workflow.
