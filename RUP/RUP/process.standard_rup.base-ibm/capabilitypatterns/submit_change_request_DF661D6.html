Task Descriptor: Submit Change Request
This task describes how to submit a change request.

Main Description

When submitting a change request, provide information that describes the change. This must include a detailed description of the requested change, at a minimum. Guidance on the type of information that should be provided when submitting a change request can be found in the Sample Change Request form.

Step 1. Identify change request.
Step 2. Document benefits, risk, and cost associated with the request.
Step 3. Log the change request in the locally adopted change request management system.
Step 4. Proceed to review and classify the change request.

Hints and tips

• Quantify known cost and benefits.
• List the effects or the impact if the change request is not executed.
• Include the time required to complete the change request.
Steps
Gather change request information

Gather the information required to describe the change request. This must include detailed description of the requested change as a minimum.

Document benefits, risk, and cost associated with the request.

Submit the change request
Submit the change request through the proper channels to ensure compliance with the established change request management process. The technique will vary depending on the tools used to manage change requests.
