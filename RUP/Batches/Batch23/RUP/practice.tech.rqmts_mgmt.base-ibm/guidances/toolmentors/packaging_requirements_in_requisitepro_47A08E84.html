Tool Mentor: Packaging Requirements in RequisitePro
Use IBM Rational RequisitePro to structure requirements using RequisitePro packages.
Tool: Rational RequisitePro
Relationships
Related Elements
  • Organize Requirements
Main Description

The RequisitePro Package structure helps you to store the project's requirements in a hierarchical structure.

To create a new package

  1. In the RequisitePro Explorer, select the project name or an existing package, and click File > New > Package.
  2. The Package Properties dialog box is displayed.
  3. In the Name box, type the name under which the package will be listed in the Explorer. The name cannot include commas, colors, nonprintable characters, or carriage returns, and it must be unique, relative to other packages at its location. The Package box contains the package that you selected in Step 1. You can change the package at any time.
  4. In the Description box, type a brief description of the package.
  5. Click OK.
  6. The new package is displayed in the Explorer, nested within the project or package that you selected in Step 1

Note: You can move the new package by dragging it where you want it to appear in the Explorer. When you do, you must be sure that the package name is unique among the other packages where it is located. You can also create the requirements or package under a package.


To delete a package

To delete a package, you must first make sure that it is empty (an empty package does not have indented artifacts listed below the package name).

  1. In the Explorer, expand the view of the package so that all of the artifacts that it contains are listed.
  2. Move or delete all of the artifacts in the package.
  3. You can drag the artifacts to other packages, or you can delete the artifacts by selecting an item in the package and clicking Edit > Delete. This removes the artifact from the project permanently.
  4. Select the package and click Edit > Delete.
  5. RequisitePro deletes the package from the Explorer, and from the project database.

To rename a package

  1. In the Explorer, select the package that you want to rename and click File > Properties.
  2. The Package Properties dialog box is displayed.
  3. In the Name box, edit the package name. The name cannot be the same as the name of another package at the current location.
  4. Click OK.
  5. The renamed package is displayed in the Explorer, sorted by the new name.
