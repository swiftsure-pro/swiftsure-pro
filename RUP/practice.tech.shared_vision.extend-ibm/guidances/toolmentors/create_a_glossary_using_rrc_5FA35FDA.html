Main Description

The Rational Requirements Composer is a collaborative toolset that supports teams to create a glossary to build consistent, common terminology for the project. Requirement context is enhanced by capturing common terminology related to requirements.

Benefits of Using Rational Requirements Composer for Glossaries

  • Capture terms that can be reused across projects and glossaries.
  • Reference any requirement artifact and link a term to it to provide context.
  • Link terms to other terms which are related or are synonyms.

Create a Glossary
Create the glossary by following the instructions in the Rational Requirements Composer Help\Creating a Glossary.

Create Terms
Terms can be independent of a glossary. They provide consistent terminology for a project. A term can also exist in one or more glossaries. Create new terms by following the instructions in the Rational Requirements composer Help\Creating Terms.

Look up Terms
Terms can be reviewed from any document in Rational Requirements Composer. Look up terms by following Help\Looking up Terms.

Link to Terms
Various requirements artifacts can be linked to a glossary to provide further information about the artifact. Link to terms by following the instructions in the Rational Requirements Composer Help\Linking to Terms.
