Artifact: Test Motivator
A motivator provides the rationale for some or all of the testing. This information can take different forms, such as a test motivator file or a test requirement.

Description
Main Description

A test motivator is usually an external file that provides details about a test plan or test case. For example, a test motivator file could be a document or spreadsheet containing detailed information about a specific test plan or test case. It is convenient to be able to reference a test motivator file from a test plan and test case record. Test motivators are useful in linking test requirements with test plans and test cases.

You can use associations between test plan and test case records with a test motivator to determine:
  • If there is test coverage for the motivator
  • If a test plan associated with a motivator has an associated external file
Tailoring
Impact of not having

This information needs to be captured in a standalone or associated artifact. If not present then you might have to deal with:

  • Lack of focus for the test effort associated with a test cycle or iteration.
  • Lack of traceability support between the test main drivers and the test scope.
  • Communication problems between stakeholders, as the test coverage could not be properly checked.
Reasons for not needing

This is a key artifact, but some organizations might capture this information within the test plan or even as part of the associated test cases.
