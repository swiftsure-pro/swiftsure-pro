Activity: Develop User Story
Start development of each assigned user story. This activity is dependent on a particular user story being identified in previous iteration or during current iteration.

Description

This activity is planned, with multiple occurrences. It is instantiated multiple times, one for each user story assigned to an iteration.

If instantiated as a work item in a tool such as Rational Team Concert, it could be created as a work item of type user story. The summary field could follow the naming convention Develop User Story: <story name>, where <story name> is replaced by the actual name of the user story.

Key Considerations

We don't show dependencies between the task to identify user stories and the tasks to develop user stories because in many cases a task to develop a user story depends on identifying that user story first but not necessarily identifying all user stories first.

Since this activity is marked as having multiple occurrences, we can't determine dependencies in the generic WBS because the real dependencies will exist between the actual instances of this activity in a real project.
