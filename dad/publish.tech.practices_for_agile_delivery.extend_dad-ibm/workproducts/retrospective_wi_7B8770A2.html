Artifact: Retrospective
This artifact is a work item type used to track the work associated with performing an iteration or project retrospective.

Description
Main Description

The iteration plan contains a section to capture assessment information. A retrospective work item is provided to track status of retrospective sessions.

Retrospectives are owned by a team lead who uses the description field of the work item and initialize it with the areas of feedback the component leads are particularly interested in. Team members add comments with their particular feedback. The team lead often updates the description to summarize that feedback. If the team lead wants to make sure everyone on the team contributes to the retrospective, he or she can create "approvals" for each team member as well.
