RUP to ITUP Capacity Management Touch Point
This page provides an inside look at unifying the RUP Test discipline with the ITUP Capacity Management process offering a heightened awareness of how software development processes affect and are affected by IT operations.
Main Description

The goal of this touch point is to set expectations for the nominal performance levels as specified in the Service Level Agreement (SLA).

When performing the  Test discipline and the task  Determine Test Results, performance test results need to be captured in the RUP artifact  Test Results in order to set expectations for the nominal performance levels as specified in the SLA.

Performance test results then need to be shared with operations to set initial thresholds for monitoring -  the ability to discover normal. Discover normal means to baseline the performance metrics for an application (and/or underlying middleware and systems) to set thresholds for operational monitors. This then sets the parameters for the ongoing measurement of the application's performance during its operation and will provide invaluable data regarding stability and bug rate and provide management with feedback on overall quality of service being delivered to the customer.

As a summary, here are the relationships between RUP and ITUP elements at touch point:

RUP Elements ITUP Element Relationship

Discipline:  Test

  • Task:  Determine Test Results
  • Artifact:  Test Results
