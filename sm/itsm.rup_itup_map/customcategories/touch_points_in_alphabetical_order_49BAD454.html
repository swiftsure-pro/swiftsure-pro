Main Description

The following table presents the method elements describing the RUP to ITUP integration touch points. Not all the relationships between RUP and ITUP, as depicted in Figure 1, are covered with touch points in this method, but may be covered in future releases.

Additionally, the following page provides access to method elements that describe the integration of RUP and ITUP from a different but complimentary perspective: Overview of the Relationship Between RUP and ITUP.

Integration Touch Point RUP Elements ITUP Elements Relationship

RUP to ITUP Analysis and Design Touch Point

Discipline:  Analysis & Design

ITUP Configuration Management Process Overview

  • Configuration Items (CIs)
  • Configuration Management Database (CMDB)
The goal of this touch point is to emphasize and elaborate on the importance of understanding the current production environment when performing architecturally related  Analysis & Design tasks.

RUP to ITUP Capacity Management Touch Point

Discipline:  Test

  • Task:  Determine Test Results
  • Artifact:  Test Results 

ITUP Capacity Management Process Overview

  • Service Level Agreement (SLA)

The goal for this integration of elements is to set expectations for the nominal performance levels as specified in the Service Level Agreement (SLA).

RUP to ITUP Change Management Touch Point


Discipline:  Configuration & Change Management

  • Task:  Establish Change Control Process; specifically its step Establish the Change Request Process
  • Task:  Verify Changes in Build

Discipline:  Requirements

  • Task:  Prioritize Use Cases
  • Task: Find Actors and Use Cases
  • Task:  Detail a Use Case
  • Task:  Detail the Software Requirements

ITUP Change Management Process Overview

  • Requests for Change (RFC)
  • Known Error
  • Configuration Item (CI)

The goal for this integration of elements is to take into account and prioritize any modification to or addition of a Configuration Item (CI) as requirements progress through application design and development.

RUP to ITUP Configuration Management Touch



Discipline:  Test

  • Task:  Define Test Environment Configurations
  • Artifact:  Test Environment Configuration

ITUP Release Management Process Overview

  • Configuration Items (CIs)
  • Configuration Management Database (CMDB)
  • Definitive Software Library (DSL)
  • Definitive Hardware Store (DHS)
The goal for this integration of elements is to ensure the test discipline derives and builds the test environment from the live production environment and its inventory/set of Configuration Items (CIs).

RUP to ITUP Problem Management Touch Point

Discipline:  Project Management

  • Task:  Develop Problem Resolution Plan
  • Task:  Handle Exceptions and Problems


ITUP Problem Management Process Overview

  • Incidents
  • Problems
  • Known Error
  • Problem Management
    • Task:  Determine Process Procedures
    • Activity: Manage Problems Proactively
The goal of this touch point is to expand RUP's coverage of the process of problem management in order to minimize the adverse impact of Incidents and Problems on the business that are caused by errors within the IT Infrastructure and to prevent the recurrence of incidents related to these errors.

RUP to ITUP Service Level Management Touch Point

Discipline:  Requirements

  • Task:  Elicit Stakeholder Requests
  • Task:  Find Actors and Use Cases
  • Task:  Detail a Use Case
  • Task:  Detail the Software Requirements
  • Task:  Develop Supplementary Specifications
  • Artifact:  Use Case
  • Artifact:  Supplementary Specifications
  • Artifact:  Software Requirements Specification
  • Artifact:  Software Requirement

ITUP Service Level Management Process Overview

  • Service Level Agreements (SLAs)
  • Service Level Management Documents

The goal for this integration of elements is to ensure the requirements for an application are fully considered and addressed within the requirements discipline by working closely with the Service Level Management (SLM) process.
