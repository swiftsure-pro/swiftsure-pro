RUP to ITUP Configuration Management Touch
This page provides an inside look at unifying the RUP Test discipline with the ITUP Configuration process offering a heightened awareness of how software development processes affect and are affected by IT operations.
Main Description

The goal of this touch point is to ensure the test discipline derives and builds the test environment from the live production environment and its inventory/set of Configuration Items (CIs).

When performing the  Test discipline and the task  Define Test Environment Configurations the test environment should reflect/mirror (perhaps in reality, perhaps through simulation) the targeted production environment and be documented using the  Test Environment Configuration artifact.

When specifying and documenting the test environment using the Test Environment Configuration artifact one needs to take into account the Configuration Management Database (CMDB), the Definitive Software Library (DSL) and the Definitive Hardware Store (DHS). Referencing the CMDB, DSL and DHS helps to insure that the integrated set of components one is planning on using to build the test environment are derived from the live production environment and its inventory/set of Configuration Items (CIs) in order to accurately replicate the live production environment.

If you do not do this you run the risk that your tested components may be successfully validated and verified but when you migrate (move) them to the live production environment you may disrupt service because a CI running in production was different than a CI that was part of your testing environment.

As a summary, here are the relationships between RUP and ITUP elements at touch point:

Discipline:  Test

  • Task:  Define Test Environment Configurations
  • Artifact:  Test Environment Configuration
