RUP to ITUP Analysis and Design Touch Point
This page provides an inside look at unifying the RUP Analysis and Design discipline with the ITUP Configuration Management process offering a heightened awareness of how software development processes affect and are affected by IT operations.
Main Description

The goal of this touch point is to emphasize and elaborate on the importance of understanding the current production environment when performing architecturally related  Analysis & Design tasks.

The current production environment documentation should provide:

  • An inventory of installed services (for example see Figure 1), applications, data, computing and network infrastructure.
  • The current status of all the non-technical, management aspects of the overall Information Technology (IT)/Information Systems (IS)  environment, including:
    • The IT/IS organization, roles and responsibilities.
    • The existing IT/IS strategy, policies and plans (including skills acquisition and sourcing options). 
    • Development policies and practices.
    • System management policies and practices.

The production environment documentation can be derived from Configuration Items (CIs) stored and managed in the Configuration Management Database (CMDB). These details about the current production environment should be summarized into a form that can be readily used as input to the  Analysis & Design tasks and infrastructure delivery and management activities, along with observations on current known IT/IS deficiencies, concerns and user satisfaction with IT.

Figure 1 - Services View

As a result the production environment work product provides the basic data and information that will be used to:

  • Provide a base point against which the IT Strategy, the Enterprise Architecture and IT based business systems can be defined.
  • Provide a basis for subsequent assessments of the current production environment for functional and technical quality.
  • Provide a basis for identifying gaps between the current environment and the future required Enterprise Architecture (and its constituent application, data and technology parts - including System Management, Network, Security, Application Development).
  • Provide a basis for subsequent assessments of the effectiveness of:
    • current IT process management, policies and plans
    • current development policies and practices
    • current accessibility policies and practices
    • system management policies and practices
    • current security and privacy policies and practices
  • Give valuable insights into possible areas of key requirements for both technical improvements to the IT infrastructure and the Application Portfolio
  • Provide a baseline for understanding the system changes necessary for new IT based business solutions

These insights will be key inputs to subsequent transition planning activities to implement the IT Strategy and Enterprise Architecture, in support of business- strategic directions and goals.
