    Define Staffing Requirements - Systems Engineering Note

    Staffing the Inception and Elaboration Phases

    In the systems engineering context, the team has similar constituents, with a System Architect as well as a Software Architect and perhaps a few more System Designers (depending on scope) because there is typically more analysis and trade study work to be done (in systems development at this level).

    The emphases are similar to software engineering, but the system architecture team is not focused solely on software and has a wider range of skills, in the systems engineering discipline itself, as well as hardware engineering and other engineering specialties.

    Staffing the Construction Phase

    At the systems level, there is a focus on integration during construction, given that, at least in a large system development, much of the actual construction might be subcontracted. So there is an intense interest in interfaces during construction, in keeping these clearly defined and stable.

    Staffing testing activities

    In addition to the issues canvassed above, there might be other factors which affect what the system test team does and its composition. There might be a considerable logistics element to test at the systems level: specialized test rigs might be needed for test performance and test analysis; testing might take place away from the development center so transport and set-up becomes an issue. The verification of specialty engineering requirements might need engineers with corresponding skills to participate, and so on.
