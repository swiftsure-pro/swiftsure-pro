Systems Engineering Note:

During systems engineering, the Flow of Events must be constructed as a sequence of steps ("black-box steps" - see  Guideline: Use Case), each of which represents a fine-grain interaction with the system. These black-box steps are used to guide the identification of "services" to be supported by the system (see Task: Define the System Context). Each black-box step might be annotated with budgeted performance or other resource constraints which are driven by external needs (for example in order to achieve some overall response time).

When doing systems engineering, the non-functional requirements or constraints are budgeted and allocated to "black-box" steps where appropriate (see  Guideline: Use Case).
