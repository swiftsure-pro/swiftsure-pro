Tool Mentor: A List of Method Composer Frequently Asked Questions
This tool mentor includes a list of Method Composer frequently asked questions and provides answers.
Tool: Method Composer
Main Description

Question: Content that I am expecting isn't showing up when I publish the configuration.
Answer: Check that the content is checked in the configuration. Whenever plug-ins, folders, or processes are added, you need to update configurations to make sure they are included.

Question: Content is not showing up in navigation views, as expected.
Answer: Confirm that the element has been added to the custom category that represents the navigation view. If the custom category is populated using a query, make sure that the element you want added has the necessary values to be included. For example, if custom category includes everything with a specific tag, make sure the element that would like to see added has been tagged with the necessary tag.

Question: I added glossary terms to a plug-in, but they are not showing up.
Answer: If the plug-in is a "supporting plug-in" then glossary terms will not show up. You may have to create both supporting and non-supporting plug-ins to separate content that publishes only when used from content that you always want published. For more information on supporting plug-ins, see the Method Composer online help topic: Product overview > Introduction to Rational Method Composer > Key concepts > Method plug-in. The latest information on this topic is also available from the RMC Information Center.

Question: I added some roles and work products in a "common" plug-in, and they are showing up in configurations where they are not applicable.
Answer: If the roles and work products are in a non-supporting plug-in, then they will show up even when not referenced by a specific practice. You may have to create both supporting and non-supporting plug-ins to separate content that publishes only when used from content that you always want published. For more information on supporting plug-ins, see the Method Composer online help topic: Product overview > Introduction to Rational Method Composer > Key concepts > Method plug-in. The latest information on this topic is also available from the RMC Information Center.

Question: My configuration contains lots of resource errors - what did I do wrong?
Answer: When elements are moved and renamed in the library, configurations are not automatically updated. Thus, over time, references in configurations can become invalid. If you are sure your library contains everything relevant to the configuration, you can apply the "quick fix" to all such errors.

Question: A field on one of my method elements (such as Purpose) is empty, but it still appears. I would like for it to be suppressed - what do I do?
Answer: It should be automatically suppressed if empty. Check the HTML tab - there is probably some invisible HTML in the field.

Question: I am trying to drag a task from the library view into my process, but it doesn't work.
Answer: You must drag from the configuration view, not the library view. 

Question: I created a task, but I can't find it to add to my process. Where is it?
Answer: Make sure the default configuration for the process includes the plug-in and package where the task is defined.
