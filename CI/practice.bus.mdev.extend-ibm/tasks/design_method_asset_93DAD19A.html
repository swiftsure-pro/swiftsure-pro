Task: Design Method Asset
Define the structure of a method asset - what its key elements are and how they are related.
Disciplines: Software Capability Improvement
Purpose
The purpose of this task is to identify and define practices and the relationships between them as well as determine the configuration for publish.

        Main Description

        Following the establishment of the a method library, this task will help in identifying and designing practices. Start with identifying the scope of this design effort that will determine whether to add a new practice or extend an existing practice.The designer can determine the practices and plug-ins to be designed.

        For each identified practice, the designer will define its method content elements, processes, and capability patterns as well as guidance and tool mentors. Categorizing the practice elements will help in organizing the practices as well as smoothing the creation of the practice plug-ins and configuration, which will wrap up this designing effort.

        Steps
        Identify scope
        Determine whether to add new practices or extend existing practices (e.g. adding or modifying plug-ins for existing practices).
        Identify practices and plugins

        Determine the kind of plug-ins to be created: including standard practices plug-ins, extension to core plug-ins, process plug-ins which identify capability patterns, and publish plug-ins which specify configurations.  

        Structure practices

        Structure a practice: 

        • Define method elements and the relationships between them.
        • Define processes and capability patterns for the practice
        • Identify guidance and tool mentors
        Categorize method elements
        Create practices, plug-ins, and configurations
        Add these method assets to the method library.
        Evaluate the design
        Confirm that the design of your method assets conform to the method library architecture. Conduct peer reviews and stakeholder reviews to identify opportunities for improvement. Revise the design as necessary.
