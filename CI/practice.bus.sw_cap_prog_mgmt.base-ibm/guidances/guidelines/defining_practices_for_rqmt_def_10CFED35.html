Guideline: Defining Practices for Requirements Definition
This guideline provides considerations for defining practices that projects will use for requirements definition.
Main Description

The following are typical activities and key considerations for Requirements Definition to reference as you define practices for your organization.

  • Describe techniques for requirements elicitation, analysis, and validation in order to obtain a shared understanding of the requirements with stakeholders.
  • Define what approaches will be used to transform stakeholder needs into customer requirements and how to allocate those requirements to product components.
  • List templates, tools and techniques that will be used by projects as they enact the requirements definition practices.
