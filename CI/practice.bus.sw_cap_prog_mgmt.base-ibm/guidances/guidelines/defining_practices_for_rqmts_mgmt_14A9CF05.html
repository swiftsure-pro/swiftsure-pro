Guideline: Defining Practices for Requirements Management
This guideline provides considerations for defining practices that projects will use for requirements management.
Relationships
Related Elements
  • Considerations for Organizational Process Definition by Process Area
Main Description

The following are typical activities and key considerations for Requirements Management to reference as you define practices for your organization.

  • Establish criteria for identifying the appropriate set of stakeholders and requirements providers
  • Establish criteria for requirements acceptance.
  • Define common attributes that will be captured for requirements.
  • Define the procedures for managing requirements changes. Describe an approach for impact analysis in order to understand the impact of requirements on existing commitments.
  • Determine how traceability will be maintained between requirements and other project work products.
  • Describe the process flow for approval and review of requirements
  • List templates, tools and techniques that will be used by projects as they enact the requirements management practices.
