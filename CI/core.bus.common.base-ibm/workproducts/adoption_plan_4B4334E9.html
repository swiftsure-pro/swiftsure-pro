Artifact: Adoption Plan
This is a comprehensive, composite artifact that gathers all information (generally of a strategic nature) required to manage the adoption project.
Domains: Software Capability Improvement
Purpose

This work product defines the strategy for rolling out process improvements to the organization.

Description
Main Description

This document serves as the implementation plan for rolling out best practices and tools to the organization. It provides schedule and resource estimates, pilot project training and mentoring estimates for deployment in support of the key business drivers for the organization.

This artifact describes all important or critical decisions regarding the proposed implementation of practices and tools. It describes the overall approach as well detailed implementation plans for deployment.
Brief Outline

This artifact typically contains the following sections:

  • Adoption goals
  • Adoption risks
  • Roll-out plan, including a list of pilot projects
  • Measurement plan
  • Hardware and software topology to support proposed improvements
  • Training and mentoring plan
  • Migration plan
  • Communication plan
  • Improvement requests
  • Process assurance review reports

Key Considerations
This artifact is updated at key stages of the roll-out project to address changing circumstances of the project and evolving requirements.
Tailoring
Impact of not having

Without this artifact:

  • planning details are likely to be poorly defined or inadequate
  • team members will not know what tasks they need to perform
  • the project will be difficult to manage and control
  • it will be hard to adopt capability improvements
Reasons for not needing

This artifact is typically required in some form on all but the smallest capability improvement projects.
