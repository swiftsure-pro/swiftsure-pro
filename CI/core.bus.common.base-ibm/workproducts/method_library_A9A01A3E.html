Artifact: Method Library
A collection of method assets, including plug-ins, configurations, and published configurations.
Domains: Software Capability Improvement
Purpose

The method library supports process improvement through the sharing of consistent approaches, standards, best practices, and lessons learned across the organization.  

It is used to store method assets and make them available to those who are defining, implementing, and managing processes in the organization.

Description
Main Description

The method library is a collection of process-related assets maintained by the organization for use by project teams. It contains method assets packaged as plug-ins, practices, and configurations. Examples include:

  • Defined processes (for example: lifecycle models and reusable capability patterns)
  • Work products descriptions
  • Role descriptions 
  • Task descriptions 
  • Guidance (for example: templates, checklists, tailoring guidelines)
