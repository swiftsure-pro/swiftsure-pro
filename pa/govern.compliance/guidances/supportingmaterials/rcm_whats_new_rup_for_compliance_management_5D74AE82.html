What's new: RUP for Compliance Management
Summarizes the plug-in's change history.
Main Description

The new features and changes from version to version are described below.

For more details on the contents, refer to Supporting Material: About RUP for Compliance Management.

Version 1.0

This version is a general rework of the earlier beta version and focuses on the following activities:

  • Refactoring the entire content base as a RUP extension
  • Presenting compliance additions as a collection of practices supporting process auditability. they Include:
    • Record Control
    • Work Product Control
    • Process Validation
    • Control Point Definition
  • Fleshing out many content elements and adding several (most notably checklists, templates, and guidelines)

It does not cover process modeling and portfolio management aspects of compliance management.

Version 1.0 Beta

This is the initial release of this plug-in.

This early beta release is focused on adding content for compliance enhancement projects. This content includes:

  • Application Compliance
     It augments today's RUP methodology to support the implementation of Application Control. Requirements are focused around compliance (for example, adding audit functionality into the application)
  • Process Compliance
     This content is focused on the development process for Compliance Enhancement Project (for example, control points and approvals for delivery of changes) .
