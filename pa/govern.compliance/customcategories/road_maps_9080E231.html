Road Maps
These two roadmaps provide a high-level view into the Rational Unified Process (RUP) for compliance management (RCM): The first one describes how process control is added to the RUP product for enhanced auditability, and the second one builds on the first one to describe how to use the enhanced RUP process to implement Application Controls.

Main Description

A key aspect of achieving Compliant status is to create confidence that internal and external policies, regulations, and standards are being implemented within a development process, and across its work products. In practice, Typical Compliance Concerns and Common Compliance Challenges can be addressed by providing legitimate evidence of their implementation to an impartial third party, also known as an auditor.

IT Compliance is managed by implementing two different types of Business Controls:

  1. IT Process Controls: These are controls that apply directly to the IT organization in order to improve the rigor of the software development process itself.
    For example, by requiring that adequate project traceability, reporting, and approval procedures are in place throughout the software lifecycle. This roadmap provides an example of IT Process Controls for a typical commercial IT software development process. These controls can be modified to reflect the level of rigor defined for a specific software development process.
    The Roadmap: Process Controls Applied to RUP describes the process controls introduced and how they are implemented inside RUP.

  2. Application Controls: These are controls implemented within corporate software. These controls are therefore software requirements, and can have regulatory or standards-based significance.
    For example, a common control that applies to the Human Resources (HR) organization is to “Limit access to classified employee data.” This control is automated by making it a common software requirement for all HR systems. The organization’s applications can thus become more consistent in bringing in the function required to provide such limited access. Because these software requirements are of interest to auditors and other compliance stakeholders, we separate out these software requirements for unique reporting and traceability processes. We will refer to the incorporation of these additional controls into existing systems as “compliance enhancement projects.”
    The Roadmap: Application Controls Development describes how the first roadmap is extended to deal with the development of application controls.
