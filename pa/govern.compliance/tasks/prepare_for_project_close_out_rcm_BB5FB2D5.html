Archive the project

At the end of a project, all of the baselined work products should be archived. An archive should include:

  • The set of artifacts to be considered for preservation for maintenance and evolution purposes
  • Information necessary to support audits

This ensures that the delivered release package can be recovered if necessary, either for audit purposes or to continue development at a later date (for example, for the next release).
