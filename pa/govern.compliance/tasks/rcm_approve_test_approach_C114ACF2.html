Task: Approve Test Approach
This task produces a formal agreement on the test approach to be taken, as defined in the test plan.
Disciplines: rup_test_discipline
Extends: Approve Work Products
Purpose

This task is closely coupled with the  Task: Agree on the Mission. It provides a formal approval of the test plan that validates the approach to be taken.

      Steps
      Review Test Approach

      The approval constitutes agreement with the test approach. This will include reviewing the objectives for the iteration, the traceability back to the compliance requirements, and the test plans. The Test Manager is responsible for this approval, but will seek input during the review from the stakeholders from the project team, including the Test Manager, the Architect, the Tester, and the Systems Analyst. This could occur through a formal review (walkthrough) of the test plan.

      Key Considerations
      Some of the compliance requirements are nonfunctional (for example, security requirements). In many projects, nonfunctional requirements tend to be mistakenly overlooked. The planned tests require special attention to ensure that they provide good test coverage of these nonfunctional requirements.
      Alternatives
      This approval may be handled more informally if this is not a specific compliance need.
