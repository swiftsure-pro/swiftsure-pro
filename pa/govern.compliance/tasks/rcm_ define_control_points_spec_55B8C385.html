Task: Define Control Points Specification
This task defines the approval of the Control Points Specification. The control points are key elements for audit purposes, because they provide controls and approvals of major artifacts and handover points.
Disciplines: rup_environment_discipline
Purpose
The purpose of defining the Control Points Specification is to clearly identify and specify the control gates imposed on the software development lifecycle.

        Main Description
        This task specifies the control points. This includes defining or modifying control points, specifying who is responsible for that task, and criteria for approval.

        Define the control points and the approval tasks within the development process
