Task: Approve Subsystem Baseline
This approval is a record of the acceptance of a subsystem baseline for general use within the development team.
Disciplines: rup_implementation_discipline
Extends: Approve Work Products
Purpose

 To ensure control over the baseline being used by the Development team.

      Main Description

      This task delineates the handover from development into the integration workspace. After a subsystem has been approved through this task, it is joined with the rest of the system code base for integration toward a system-level baseline. In an automated environment, this approval opens the gate to make the changed set of work products available in the integration workspace.

      The chief concern for the Integrator is to review the content of the baseline. He or she is predominantly concerned that the baseline is of appropriate quality and stability to be approved as the common baseline from this point, forward. This is a significant control point, because the baselined set of work products becomes the default for the Development team.

      If the Integrator identifies any issues with the baseline, the associated Developers need to be informed. If the issues are a serious concern, then a corrective plan needs to be put in place. The Integrator or Technical Lead may choose not to approve the baseline until the issues have been addressed

      Steps
      Verify entry criteria
      Verify that all prerequisites that are subject to approval for the completion of the work products subject to approval have been met.
      Inspect work product and supporting material
      This step is not a thorough review; it is closer to spot-checking. It should be supported by an appropriate checklist
      Engage concerned parties to resolve concerns

      If the stakeholders identify any issues with the compliance approach, the approver will need to review these with the project team. If the issues are a serious concern, a corrective plan should be put in place. The Approver may decide to postpone or deny approval of the work products until the issues have been agreed upon and addressed.

      Create approval record
      The Approver signs off the changes either by signing a paper copy of the work product or uses an electronic signature when the tooling is available. For electronic signatures, the work order associated with the work product or an electronic reference to the work product is approved by the person in the role that is responsible for this task.
