Description
Main Description

Creating a baseline for a set of work products implies that a significant goal or milestone has been reached. Baselines can occur at many levels, from basic components to subsystems of increasing complexity, to system-level baselines.

Not all changes to a project repository cause the creation of a baseline. To be meaningful, a baseline must include references to the following information:

  • A meaningful name or label
  • The previous baseline that it is based on
  • The overall goal that it achieves and the change requests that it fulfills
  • The level of testing that it has been subjected to, including any available test logs.
