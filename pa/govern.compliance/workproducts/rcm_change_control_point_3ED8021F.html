Outcome: Change Control Point
This control point provides a gate where new changes are approved for delivery to a shared team integration stream
Domains: implementation
Work Product Kinds: Record
Extends: Control Point
Purpose

To formally control the integration of a set of changes created by a team of developers.

Main Description

The change Control Point acts as the gatekeeper for the delivery of changes from a development workspace to a subsystem integration workspace. This approval opens the gate to make the changed set of work products -- in a private work area -- available  for subsystem  integration, to produce a subsystem-level baseline.

This approval is also referred to as the Change Delivery control point.

Key Considerations

Strongly consider tooling opportunities to securely automate the creation of records and store the resulting record, particularly if the project is large or if the team is geographically distributed.

Tailoring
Reasons for not needing

Each organization will need to refine the set of records that it requires to align with its compliance needs and development process. Records may be added or removed in accordance with an organization's policies or process results.
