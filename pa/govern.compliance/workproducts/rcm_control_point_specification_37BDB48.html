Description
Main Description

The control points and approvals referenced in the Control Point Specification are essential for audit purposes, because they provide definitions of the controls and approvals of major artifacts and handover points.

The control points created in this plug-in are defined in the Example: Control Point Specification for RUP for Compliance Management. They include these control points::

  • Change Control Point
  • Baseline Control Point
  • Release Control Point

For each control point, the approval criteria are clearly documented. The information contained in this work product is a part of the  Artifact: Development Process. The reason that it has been separated into a sub-artifact is that its content is of critical importance in documenting compliance to an Auditor. This specification can be used to describe the quality control approach to an Auditor. It is also provided as input to the Configuration Management Plan.

Key Considerations

Organizations typically produce a standard Control Point Specification and a supporting environment from a central team. This is provided to the project team as a standard, where it might be used as-is or tailored as part of the development case tasks.
