Main Description

To be successful in complying with standards and regulations, you must validate that the security mechanisms work.

Authentication

Authentication is the process that validates the user. The key aspects of password authentication to consider include:
  • Each user must have a unique user ID and password
  • Each user requires a periodic password change
  • Passwords must include letters and numbers
  • Passwords must be encrypted
  • Password rules must be enforced

Access

Access refers to the ability to create, read, modify, or delete data. The key aspects to consider for access include:
  • Timely removal of terminated users from the system
  • Appropriate permissions for users, systems, and artifacts
  • Automatic logoff after a certain period of time

Approvals

Approvals are the method by which acceptance is sanctioned. Approvals are put in place to:
  • Prevent introduction of unapproved changes into the business system (or software)
  • Ensure accountability (non-repudiation)
  • Require electronic signatures or paper sign-offs
