Main Description

This specification must be adjusted for each project, because certain records prescribed by the process may not be applicable for a particular implementation. Specifically, it must contain the items described here.

  • An organized list of the records that the process must produce, listing:
    • Mandatory properties for each record
    • Any requirements that the creation of these records is subject to (for example, that a certain approval must be based on a formal review)
    • Separate categories for:
      • work product control records
      • test control records
      • process metrics (if any)
  • Ideally, the specification of each record type should include a description of how it will be created, stored, and accessed
  • A record retention and disposition table stating the length of time that each kind of record must be kept
The specification will also include, where necessary, any special requirement regulating the production of these records (which could include ownership, frequency, and so on).
