Main Description

Compliance concerns

  • The main impact on the responsibilities of the Implementer lies in the environment in which they develop. It is a closed environment, where control points are imposed as key events in the development workflow. These control points provide checkpoints that allow or prohibit continuation.

    Example control point: The Technical Lead must approve new changes by developers before delivery to the shared team integration workspace.
  • The Implementer is also responsible for translating the design specification into the software product or application. From a compliance perspective, this results in developing Application Control.
