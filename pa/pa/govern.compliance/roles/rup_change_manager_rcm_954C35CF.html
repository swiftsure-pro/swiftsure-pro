    Main Description

    Compliance concerns

    The key aspects of compliance that have an impact on the responsibilities of the Change Manager include:

    • Clearly defined control points are imposed as key events in the development workflow. These control points provide checkpoints that allow or prohibit continuation. These control points need to be clearly defined within the Artifact: Change Workflow Specification as part of the  Artifact: Configuration Management Plan. The Change Manager is responsible for this task, along with the automation of these control points.

      Example control point: The Technical Lead must approve new changes by developers before delivery to the shared team integration stream.

    • Works alongside the Release Manager to define the software development change process and the link into release management

    • Participates on the Change Control Board (CCB) alongside the Release Manager and the Project Manager.
