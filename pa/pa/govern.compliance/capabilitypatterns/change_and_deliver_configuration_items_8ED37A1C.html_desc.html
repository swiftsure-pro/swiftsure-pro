Activity: Change and Deliver Configuration Items
This activity manages project artifacts and the work involved, from their initial creation as private artifacts through to their delivery and general availability to the project team and other stakeholders.

Description

This activity is focused on:

  • Creating workspaces, accessing project artifacts, making changes to those artifacts, and delivering the changes for inclusion in the overall product (by any role in the project team).
  • Building the product, creating baselines, and promoting the baselines for availability to the rest of the development team.

Staffing

Any Role is used to represent any one of the roles defined in this configuration of the IBM® Rational Unified Process® (RUP®), where someone playing that role may want to make changes to a RUP artifact (configuration item).

The Integrator (as described in other activities and in Roles) needs to be sure that artifacts delivered from the developer workspaces are sufficiently tested so that they can be incorporated into a testable build. The Integrator needs to be familiar with Project Configuration Management (CM) Policies and Test Practices.

Usage
Usage Guidance

Typically, this work starts in the Inception phase and continues throughout the lifecycle. This work is not considered optional.
