Task: Approve Delivered Changes
This approval provides a sign-off point for the delivery of changes into the integration workspace. This approval is also referred to as the Change Delivery control point.
Disciplines: rup_implementation_discipline
Extends: Assess Control Point
Purpose

 To produce a formal approval of the changes and approval of their delivery into the integration stream.

      Main Description

      This task is closely coupled with the Task: deliver_changes_rcm.

      The Technical Lead reviews the changes to be delivered. That person is predominantly concerned that the changes are of appropriate quality to be released into the integration workspace. To approve the changes, the Technical Lead signs off on the changes either by signing a paper copy of the changes or by electronic signature. For electronic signatures, the work order associated with this change is approved by the person in charge of this task. In an automated environment, this approval opens the gate to make the changed set of work products available in the integration workspace.

      If the any issues with the changes are identified during this review, the Technical Lead will discuss these with the Implementer responsible for these changes. If the issues are a serious concern, then a corrective plan needs to be put in place. The Technical Lead will choose not to approve the changes into the integration workspace until the issues have been addressed.

      Steps
      Inventory and spot-check input work products
      Review decisions and problem resolutions
      Assess overall results
      Address any concerns with responsible principals
      Create Approval Record
