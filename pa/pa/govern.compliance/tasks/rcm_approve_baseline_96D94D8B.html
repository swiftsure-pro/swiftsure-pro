Task: Approve Baseline
This task formally identifies an official system baseline, thus it implements the Register Derived Objects control point.
Disciplines: rup_config_and_change_manage_discipline
Extends: Assess Control Point
Purpose

To ensure control over the promotion of a set of changes into integration.

      Main Description

      This task is a precursor to Task: promote_baseline_rcm.

      The Release Manager inspects the baseline and its content. This involves examining the enclosed change requests to ensure that they have gone through the correct change process and have been tested. The manager is predominantly interested in ensuring the quality of the baseline. For electronic signatures, a record of the release provides the details of the release content and the enclosed change requests.

      For the build to pass this gate, the source code must be under version control, and the build must have occurred in a controlled environment. This is a significant control point, because the baselined set of work products moves into QA or production.

      If the Release Manager identifies any issues with the baseline, that manager needs to communicate with the Technical Lead and the Integrator. If the issues are a serious concern, then a corrective plan needs to be put in place. The Release Manager will choose not to approve the baseline until the issues have been addressed.
