Task: Define Process Validation Plan
This task defines a plan to verify the correct operation of the processes governing software development.
Disciplines: rup_project_management_discipline
Purpose

The purpose of this task is to define a software development Process Validation plan that specifies how to test the development process.

      Main Description

      The Process Validation objectives are to provide "reasonable assurance" of the correct implementation and operation of the process. This validates that the process works and satisfies applicable compliance requirements, policies, standards, and regulations.

      Steps
      Review control and approval points
      Review the requirements to be tested

      For each step of the process, write a test that:

      • Examines the "happy" path.
      • Tests exceptions to the process. For each exception, validate that the error and exception are logged. Auditors look for handling of exceptions to the process. A process that ran for 6 months with no exceptions is as suspicious as a project that runs for 6 months with all milestones completed on time and on budget. Thus, you need to validate the exception handling.
      • Validates that audit trails for artifacts are enabled:
        • Validate that all data and updates appear on the records
        • Validate that records cannot be accessed without valid IDs and passwords
      • Validates that electronic signatures are valid and enabled:
        • If signature is invalid, verify that the control point is not passed
        • Validate that security logs exist and are updated for invalid passwords
      • Validates that metrics exist, and are updated and correct.
      • Checks all of the steps of the process.
      Capture the approach
      Capture the defined tests in the Software Development Validation plan.
      Key Considerations

      The organizational may use a standard software development validation plan that is given to each project team to use as a base.

      The plan should be written, so that it can be given to an audit team upon request.
