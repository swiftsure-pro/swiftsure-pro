Task Descriptor: Implement Method Asset
This task is where the individual method elements are detailed (i.e., the text is written), and one or more publishable configurations are created.

Main Description
This task follows the design of a method element: filling in details about this element based on its type (work product, task, role, etc.). Testing at element level after initial implementation and regression test on the entire library until the result is satisfactory (usually error free for all configurations). Then make sure the changes are checked into the library repository if applicable.
Steps
Detail method elements

Method elements include common fields (such as purpose and main description, etc.), practice elements, work products, tasks, guidance, roles, tool mentors, and processes.

Document critical attributes, such as entry and exit criteria, for method elements in associated templates and checklists.

Test and fix errors
Check-in the changes to the method library
