Guideline: Creating a Block Definition Diagram
This guideline describes naming and drawing conventions for block definition diagrams.

Main Description

Guidelines and Drawing Conventions

  • Keep block names simple within structure diagrams, even when they are referencing blocks across packages.
  • Blocks should not show attributes, operations or ports.
  • The stick figure shall only be used to visualize actors that are external to the system. Actors that represent subsystems shall be shown as blocks.

Naming Conventions

  • The name of a block definition diagram shall have the pre-fix "BDD_".
