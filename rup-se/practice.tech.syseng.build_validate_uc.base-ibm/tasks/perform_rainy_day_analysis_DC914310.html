Task: Perform Rainy-Day Analysis
This task focuses on the identification of system error / fail behavior that was not covered in the initial set of requirements.
Disciplines: System Engineering Requirements
Purpose
To capture the exception behavior and to verify and validate the extended use-case model.
Relationships
RolesPrimary Performer:
  • Requirements Engineer
Additional Performers:
InputsMandatory:
  • System Use Case
  • System Use-Case Activity Diagram
  • System Use-Case Internal Block Diagram
  • System Use-Case Sequence Diagram
  • System Use-Case Statechart Diagram
  • System-Wide Requirements
    		Optional:
    • None
    Outputs
    • System Use-Case Activity Diagram
    • System Use-Case Internal Block Diagram
    • System Use-Case Sequence Diagram
    • System Use-Case Statechart Diagram
      Steps
      Identify exception behavior
      Extend Use-Case Block Statechart Diagram
      Update Use-Case Block-Box Activity Diagram (if needed)
      Update Use-Case Internal Block Diagram (if needed)
      Verify/Validate Use-Case Model through execution
      Record exception scenarios
