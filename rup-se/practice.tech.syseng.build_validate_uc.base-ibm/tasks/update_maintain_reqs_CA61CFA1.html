Task: Update and Maintain Requirements
This task establishes traceability from the Use-Case block properties to the relevant system requirements.
Disciplines: System Engineering Requirements
Purpose
To establish the traceability to support coverage, completeness and impact analysis.

        Main Description

        This task establishes traceability from the use-case block operations and attributes to the system requirements that they implement.  This traceability information is initially used to ensure that each system requirement has been considered and is implemented by at least one use-case block (coverage analysis).  Later in the lifecycle this traceability information is used to assess the impact of proposed changes on the solution architecture.

        Steps
        Identify system requirements that are applicable to the subject system use case
        Review the system requirements specification and the traceability record to identify the functional and non-functional requirements that apply to the subject use case.  This information should be available in the Traceability Record as a result of previous work done detailing the system requirements.
        Establish traceability from the relevant use-case block properties to the system requirements
        Based upon the system requirements that are relevant to the subject use case, create <<satisfies>> links from each of the subject use-case block operations and attributes to the associated system functional and non-functional requirements, respectively.
