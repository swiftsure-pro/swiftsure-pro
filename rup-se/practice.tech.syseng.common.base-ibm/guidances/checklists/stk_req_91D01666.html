Checklist: Stakeholder Requirement
This checklist provides guidance on assessing the quality of requirements.
Extends: General Requirements
Relationships
Related Elements
  • Analyze and Refine Stakeholder Requirements
  • Stakeholder Requirements Specification
  • Systems Requirements Specification
Check Items
The requirement is described in the stakeholder's language
Does the requirement express a stakeholder's need, in the stakeholders' own words?
The requirement is prioritized
Is there a priority attribute (high/medium/low, must/should/could/would, or equivalent) set for the requirement?
The requirement has a rationale
Is there a description of why the stakeholders need it (background, context, reasons)?
