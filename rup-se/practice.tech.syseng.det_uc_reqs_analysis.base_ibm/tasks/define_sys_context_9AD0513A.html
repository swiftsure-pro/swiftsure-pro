Task: Define System Context
This task describes how to a create a system context diagram that shows the top-level collaboration between the system and its actors.
Disciplines: System Engineering Requirements
Purpose
Based on the Use-Case Model,  to create a top-level collaboration showing the system (modeled as a top-level subsystem), its interfaces and its relationships with its actors, including the external I/O entities that flow between actor and system.

      Steps
      Create initial Context Diagram
      Create an initial context diagram showing the top-level collaboration between the system and its actors with the system interfaces abstracted away. For more information, see Defining the System Context.
      Refine associations and interfaces
      Start reasoning about the system operations and its attributes. Add interfaces and identify the I/O entities and start to characterize the connections between the actors and the system. For more information, see Defining the System Context.
      Details system operations and other characteristics
      Start to construct scenarios as a base for describing system operations. Identify the required and the provided interfaces. For more information, see Defining the System Context.
