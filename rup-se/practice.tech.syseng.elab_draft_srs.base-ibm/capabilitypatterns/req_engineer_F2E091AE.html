Role Descriptor: Requirements Engineer
This role leads and coordinates requirements elicitation and documents and maintains the different levels of stakeholder and system requirements.
Based on Method Role: Requirements Engineer
Relationships
Performs:
  • Analyze and Refine Stakeholder Requirements
  • Generate System Requirements
  • Identify System Use Cases
  • Link Stakeholder Requirements to System Requirements
  • Prioritize and Partition Use Cases
  • Trace Use Cases to System Requirements
Responsible for:
  • Glossary
  • Interface Control Document
  • Stakeholder Requirements Specification
  • System Context Diagram
  • System Operation
  • Systems Requirements Specification
  • System Use Case
  • System Use-Case Activity Diagram
  • System Use-Case Diagram
  • System Use-Case Internal Block Diagram
  • System Use-Case Priority List
  • System Use-Case Sequence Diagram
  • System Use-Case Statechart Diagram
  • System-Wide Requirements
  • Traceability Record
  • Traceability Record [Non-functional Requirements to Stakeholder Requests]
  • Traceability Record [System to Stakeholder Requirements]
  • Traceability Record [Use Cases to Stakeholder Requests]
  • Traceability Record [Use Cases to Systems Requirements]
Additionally Performs:Modifies:
  • Glossary
  • Interface Control Document
  • Stakeholder Requirements Specification
  • Systems Requirements Specification
  • System Use Case
  • System Use-Case Diagram
  • System Use-Case Priority List
  • Traceability Record [System to Stakeholder Requirements]
  • Traceability Record [Use Cases to Systems Requirements]
