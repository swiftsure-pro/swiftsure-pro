Task: Define Candidate Solutions
This task focuses on identifying possible solutions for a chosen key system function.
Disciplines: System Engineering Architecture
Purpose
There is always more than one way to realize a key system function. The objective of this stage is to identify possible solutions for each previously identified key system function.

      Main Description
      Each key system function is analyzed with regard to its realization. At this stage, associated stakeholder requirements need to be identified and taken into consideration. Typically, first-cut design decisions are made: "Buy or build?" Should the key system function be realized as a COTS component or developed internally? Based on these preconditions, possible solutions then are identified in a team representing all relevant areas of expertise. The task ends with a set of agreed alternative solutions for the chosen key system function.
      Steps
      Choose a key system function
      Determine the context of the solution by selecting a key system function previously identified.
      Analyze the key system function
      Identify the system requirements and system functions that are associated with the subject key system function. Analyze the collaboration of its constituents (system functions). Identify associated non-functional requirements (i.e. design constraints).
      Identify candidate solutions
      Elaborate possible solutions in collaboration with a multidisciplinary team of domain experts. Consider and identify previously developed hardware and software components, non-developmental items, and COTS hardware and software as candidates.
      Select candidate solutions
      Identify those solutions that will be considered as candidates for a detailed trade study analysis (Weighted Objectives Analysis).
      Create Trade Study Report
      Capture the selected candidate solutions in the Trade Study Report.
